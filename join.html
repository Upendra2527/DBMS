Join

  • Theta Join (⨝): The theta join combines tuples from two relations based
    on a given join condition that uses comparison operators
    (e.g., equality, greater than, etc.).
    Example: R ⨝_{condition} S


  • Natural Join (⨝): The natural join combines tuples from two relations based on the equality
    of attributes with the same name in both relations. It automatically matches attributes with the
    same name and eliminates duplicate columns in the result.
    Example: R ⨝ S


  • Equijoin (⨝): Equijoin is a specific type of theta join where the join condition is based on
    equality of attributes.
    Example: R ⨝_{R.attr = S.attr} S


  • Left Outer Join (⟕): The left outer join returns all tuples from the left relation and the matching
    tuples from the right relation. If there is no match in the right relation, NULL values are used for the
    right relation's attributes.
    Example: R ⟕ S


  • Right Outer Join (⟖): The right outer join returns all tuples from the right relation and the
    matching tuples from the left relation. If there is no match in the left relation, NULL values are used
    for the left relation's attributes.
    Example: R ⟖ S


  • Full Outer Join (⟗): The full outer join returns all tuples from both the left and right relations
    and fills in NULL values for non-matching attributes.
    Example: R ⟗ S
